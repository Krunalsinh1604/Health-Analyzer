# Machine Learning Architecture - Health Analyzer

This document provides a comprehensive overview of how Machine Learning is integrated into the Health Analyzer application. It is designed to demonstrate the flow of data from the user interface through to the trained models, highlighting exactly where predictive algorithms are used.

## 1. System Data Flow

The Health Analyzer uses a microservices-style architecture to process predictions:

1.  **Frontend (React):** The user inputs their vitals or uploads a report (e.g., on the Dashboard or Heart Disease page). React sends this data as a JSON payload to the FastAPI backend.
2.  **Backend API (FastAPI):** Endpoints such as `/predict`, `/predict/heart`, `/predict/hypertension`, and `/cbc/analyze` receive the data.
3.  **ML Service (`ml_service.py`):** The API passes the parsed data into `src.ml_service.py`. This service converts the Python dictionary into a `pandas.DataFrame`.
4.  **Trained Models (Scikit-Learn):** The DataFrame is passed to a pre-trained machine learning model (loaded via `joblib` from `.pkl` files stored in `backend/models/`).
5.  **Output Generation:** The model outputs a class prediction (e.g., 1 for Positive, 0 for Negative) and a probability score (e.g., 0.85).
6.  **Response:** The prediction and probability are returned to the React frontend in the `ml_model_insights` JSON object and displayed to the user.

## 2. Implemented ML Models

The project utilizes four distinct Machine Learning models, all trained on clinical datasets using the Scikit-Learn library.

### A. Diabetes Prediction Model
*   **Algorithm:** Logistic Regression
*   **Model Source:** `backend/models/diabetes_model.pkl`
*   **Training Script:** `backend/src/train_model.py`
*   **Input Features:** \`Glucose\`, \`BloodPressure\`, \`SkinThickness\`, \`Insulin\`, \`BMI\`, \`DiabetesPedigreeFunction\`, \`Age\`
*   **Output:** Binary classification (1 = Positive for Diabetes, 0 = Negative).

### B. Heart Disease Prediction Model
*   **Algorithm:** Random Forest Classifier (or equivalent ensemble model depending on latest training run)
*   **Model Source:** `backend/models/heart_model.pkl`
*   **Training Script:** `backend/src/train_models.py`
*   **Input Features:** \`age\`, \`sex\`, \`cp\` (chest pain type), \`trestbps\` (resting blood pressure), \`chol\` (cholesterol), \`fbs\` (fasting blood sugar), \`restecg\` (resting electrocardiogram), \`thalach\` (max heart rate), \`exang\` (exercise induced angina), \`oldpeak\`, \`slope\`, \`ca\` (number of major vessels), \`thal\` (thalassemia).
*   **Output:** Binary classification (1 = High Risk of Heart Disease, 0 = Low Risk).

### C. Hypertension Prediction Model
*   **Algorithm:** Random Forest Classifier
*   **Model Source:** `backend/models/hypertension_model.pkl`
*   **Training Script:** `backend/src/train_models.py`
*   **Input Features:** \`age\`, \`sex\`, \`bmi\`, \`heart_rate\`, \`activity_level\`, \`smoker\`, \`family_history\`
*   **Output:** Binary classification (1 = High Risk of Hypertension, 0 = Low Risk).

### D. Complete Blood Count (CBC) Analysis Model
*   **Algorithm:** Random Forest Classifier
*   **Model Source:** `backend/models/cbc_model.pkl`
*   **Training Script:** `backend/src/train_models.py`
*   **Input Features:** \`Hemoglobin\`, \`RBC\`, \`WBC\`, \`Platelets\`, \`MCV\`, \`MCH\`, \`RDW\`, \`Neutrophils\`, \`Lymphocytes\`, \`Monocytes\`, \`Eosinophils\`, \`Basophils\`
*   **Output:** Multi-class classification (Predicts specific potential conditions such as Anemia, Infection, Normal, etc.).

## 3. Transparency & Evaluation

To ensure the system's decisions are transparent, the backend now explicitly returns the exact algorithm used and the statistical confidence (probability) of the prediction to the UI.

This means that instead of a black-box "High Risk" label, the doctor or user can see that the prediction was generated by a "LogisticRegression" model with an "87.5%" confidence score, allowing for better clinical decision support.
